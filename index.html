01 - Incident

What happened

A packaging mistake exposed what appears to be a large part of Anthropic's most commercially valuable coding product. The code leak and the parallel supply-chain risk were separate events, but they collided on the same day.

March 31, 2026 - Around 00:00 UTC

The build ships

Claude Code version 2.1.88 lands on npm with a large cli.js.map file that appears to embed original TypeScript source through sourcesContent.

Around 00:21 to 03:29 UTC

Supply-chain attack window

Separate from the leak itself, malicious axios versions reportedly hit npm with a remote-access trojan during the same window.

Around 04:23 AM ET

Public discovery

Security researcher Chaofan Shou posts about the exposed archive on X, and the story spreads fast enough that containment effectively ends there.

Within hours

The mirrors multiply

GitHub mirrors, clean-room rewrites, architecture analysis, and prompt archaeology start almost immediately.

Anthropic response

Confirmation and takedowns

Anthropic frames the event as a release packaging issue caused by human error and begins takedown requests, but the archive is already distributed.

Possible root cause: the report ties the leak to Bun source-map behavior. If that holds, a toolchain assumption became a global disclosure event.

Why this story is bigger than one bad publish: it exposed architecture, roadmap clues, and attack surface all at once.

04 - What to do now

Action items

Different readers need different next steps. These are the practical moves that matter most right now.

If you updated Claude Code through npm on March 31

Audit for compromised dependency versions and treat any hit as a serious incident. Rotate secrets, inspect the host, and do not assume the source leak was the only thing you pulled.

npm ls axios plain-crypto-js rg -n "1\.14\.1|0\.30\.4|plain-crypto-js" package-lock.json yarn.lock pnpm-lock.yaml bun.lock

If you are trying to build or inspect the leaked source

Verify every dependency before you install anything. Internal package names becoming public is exactly the kind of moment attackers use to seed believable fakes.

If you build competing agent products

The architecture is worth studying, but a clean-room implementation is the safer path. The more your work looks independent, the lower the legal and reputational drag.

If you are just a regular user following the story

The report does not claim customer conversations or model weights were exposed. The bigger issue is what the leak reveals about product direction, operational discipline, and future attack research.
