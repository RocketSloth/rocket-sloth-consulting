01 - Incident Timeline

What happened

A packaging mistake exposed 512,000 lines of Anthropic's most commercially valuable coding product to the whole world.

~00:00 UTC, March 31

Claude Code 2.1.88 is pushed to npm

It includes a 59.8 MB source map file named cli.js.map with a sourcesContent field embedding every original TypeScript source file. In practice, that turns one package artifact into a code archive.

~00:21-03:29 UTC

Unrelated supply-chain attack hits the same window

Malicious versions of axios (1.14.1 and 0.30.4) containing a Remote Access Trojan are published to npm. Anyone installing Claude Code during that window may have pulled the compromised dependency.

~04:23 AM ET

Chaofan Shou posts the discovery on X

Security researcher Chaofan Shou (@Fried_rice) posts the discovery publicly with a download link to the archive on Anthropic's own R2 storage. The post goes viral.

Within hours

The archive spreads everywhere

The codebase is mirrored across GitHub, reportedly crossing 84K+ stars and 82K+ forks. Clean-room rewrites begin in Python and Rust while developers start dissecting prompts, flags, memory systems, and internal benchmarks.

Anthropic response

Confirmation and takedowns

Anthropic confirms the leak, saying: "This was a release packaging issue caused by human error, not a security breach." DMCA requests follow, but the source is already permanently in the wild.

Root cause callout: Claude Code is built on Bun, which Anthropic acquired in late 2025. A Bun bug filed March 11, 2026 reported source maps being served in production despite documentation implying otherwise. Anthropic's own toolchain likely helped cause the exposure.

Second leak callout: This was Anthropic's second exposure in one week. Days earlier, internal documents about an upcoming model were reportedly found accessible through a public CMS cache.

02 - Reporting Impact

Top 10 findings

These are the details most likely to keep echoing through AI product strategy, developer tooling, and security research.

01
Ethics

Undercover mode

File undercover.ts reportedly strips all traces of AI involvement when Claude Code contributes to public open-source repositories. The system prompt says: "You are operating UNDERCOVER in a PUBLIC/OPEN-SOURCE repository. Do not blow your cover." It is a one-way door: it can be forced on, but not forced off.

Ethics / transparency
02
Roadmap

KAIROS - autonomous daemon mode

Referenced 150+ times, KAIROS points toward an always-on background agent with autoDream memory consolidation, daemon workers, GitHub webhook subscriptions, 5-minute cron refresh cycles, and push notifications. It is heavily feature-gated, but the scaffolding appears complete.

Product roadmap
03
Bench

Internal model benchmarks and codenames

The leak reportedly maps Capybara to Claude 4.6, Fennec to Opus 4.6, and Numbat to an unreleased model. It also exposes that Capybara v8 had a 29-30% false claims rate, up from 16.7% in v4, along with an "assertiveness counterweight" to tame overly aggressive refactors.

Competitive intel
04
Shield

Anti-distillation mechanisms

Layer one uses the ANTI_DISTILLATION_CC flag to inject decoy tool definitions into the system prompt. Layer two summarizes assistant text between tool calls and signs the summaries cryptographically instead of returning the raw chain. Both are reportedly easy to bypass, which makes the real barrier look legal rather than technical.

Security / IP protection
05
DRM

Native client attestation

Requests reportedly include a cch=00000 placeholder that Bun's Zig HTTP stack overwrites with a computed hash before the request leaves the process. That makes the attestation invisible to JavaScript tooling and turns it into API DRM enforced below the JS layer.

Security / access control
06
Memory

Self-healing memory architecture

The reported three-layer memory design uses MEMORY.md as a lightweight pointer index, project notes for deeper knowledge, and selectively searched session history. The key design decision is skeptical memory: the agent treats memory as a hint and verifies against the real codebase.

Architecture
07
Threat

Concurrent supply-chain attack

The axios RAT was only part of it. Attackers also began typosquatting internal npm package names revealed in the leak, such as color-diff-napi and modifiers-napi, targeting developers trying to compile or inspect the leaked source.

Active threat
08
Ops

250,000 wasted API calls per day

A code comment in autoCompact.ts reportedly says: "1,279 sessions had 50+ consecutive failures (up to 3,272), wasting ~250K API calls/day globally." The fix was reportedly three lines of code: cap consecutive failures at 3.

Operational
09
Regex

Frustration detection via regex

A regex in userPromptKeywords.ts matches profanity and anger expressions to detect user frustration. It is cheaper and faster than running a sentiment model, but it raises obvious questions about behavioral telemetry and how that signal is used.

Privacy / telemetry
10
Root

The Bun bug that caused it all

Bun bug reports from March 11, 2026 said source maps could be served in production despite the docs. Anthropic had acquired Bun only months earlier. If that bug is the root cause, Anthropic's own recently acquired toolchain helped expose its own flagship product.

Root cause

04 - Action Items

What to do now

Different audiences need different next steps. These are the ones that matter most right now.

If you updated Claude Code via npm on March 31

Check your lockfiles for axios 1.14.1, 0.30.4, or plain-crypto-js. If found, treat the machine as compromised, rotate every secret, and do a clean OS reinstall. Migrate to Anthropic's native installer.

grep -r "1.14.1\|0.30.4\|plain-crypto-js" package-lock.json grep -r "1.14.1\|0.30.4\|plain-crypto-js" yarn.lock grep -r "1.14.1\|0.30.4\|plain-crypto-js" bun.lockb

If you're trying to build the leaked source

Do not install unfamiliar npm packages that claim to be Claude Code dependencies. Typosquatting is already active. Verify every dependency against the actual leaked package.json before installing anything.

If you're building competing agent tools

The architectural patterns are worth studying: skeptical memory, prompt orchestration, cache economics, and layered bash security. But Anthropic is issuing DMCA takedowns, so clean-room implementations are the safer legal path.

If you're a regular Claude Code user

No customer data, credentials, or model weights were reported as exposed. The tool itself is unchanged. If you installed via the native installer, no action is needed.
